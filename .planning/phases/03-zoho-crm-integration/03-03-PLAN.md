# Plan 03-03: Pipeline Integration & Admin Queue Sync

Integrate the Zoho synchronization service into the main daily processing pipeline and establish the sync for system errors to the Zoho Admin Dashboard.

## Problem Statement

The Zoho synchronization must happen automatically as part of the daily 3:00 AM pipeline, immediately following the scraper and business logic engine. Furthermore, to provide a single pane of glass for administrators, all `SystemError` records captured during processing must be synced to the `Admin_Queue` custom module in Zoho.

## Proposed Changes

### 1. Daily Pipeline Integration (`src/processor/pipeline.py`)
- Call `sync_all(session)` as the final step in `run_post_scrape_pipeline`.
- Ensure Zoho sync failures are logged locally and do not cause the entire pipeline to abort.

### 2. Admin Queue Sync (`src/integrations/zoho/sync_service.py`)
- Implement `sync_errors(session)` to push `SystemError` records to the `Admin_Queue` module.
- Mark `SystemError` records as "synced" locally once they reach Zoho.

### 3. Main Entrypoint (`src/main.py`)
- Ensure environment variables are loaded for Zoho CRM before starting.
- Add error handling around the overall sync process.

## Task Breakdown

- [ ] **Task 1: Implement Admin Queue Sync**
