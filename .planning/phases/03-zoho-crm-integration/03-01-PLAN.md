# Plan 03-01: Data Model Updates & OAuth Client

Establish the database foundations and the secure authentication layer for Zoho CRM communication.

## Problem Statement

The current database models lack the `zoho_record_id` fields required to track sync status between the local database and Zoho CRM. Additionally, we need a secure, reusable OAuth2 client to handle access token refreshes for the Zoho API.

## Proposed Changes

### 1. Database Models (`src/database/models.py`)
- Add `zoho_id: Optional[str] = Field(default=None, index=True)` to:
    - `Project`
    - `Company`
    - `Bid`
- Add a new `ZohoSyncState` table to track when the last sync occurred.
- Add `zoho_id` to `SystemError` (or use a separate table for Admin Queue if needed, but PRD implies syncing `SystemError` to `Admin_Queue` module).

### 2. Zoho OAuth Client (`src/integrations/zoho/client.py`)
- Implement `ZohoClient` class to handle:
    - Token refresh logic using `ZOHO_CLIENT_ID`, `ZOHO_CLIENT_SECRET`, and `ZOHO_REFRESH_TOKEN`.
    - In-memory caching of the access token.
    - Base `_request` method with automatic retry on 401 (refresh and retry).

### 3. Environment Configuration (`.env.example`)
- Add Zoho credential placeholders.

## Task Breakdown

- [ ] **Task 1: Update Database Models**
    - Add `zoho_id` to `Project`, `Company`, `Bid`, and `SystemError`.

- [ ] **Task 2: Implement Zoho OAuth Client**
